📌 Resumen del Proyecto

El proyecto implementa flujos de automatización web y raspado (scraping) inteligente utilizando Stagehand, un framework de automatización de nivel superior desarrollado por Browserbase.

A diferencia de Playwright o Selenium clásicos que dependen de selectores de CSS o XPath rígidos y fáciles de romper ante cualquier rediseño, Stagehand utiliza Modelos de Lenguaje Avanzados (LLMs, en este caso Google Gemini) para interpretar páginas de manera semántica, realizar clics, rellenar formularios, observar contenidos y extraer datos estructurados directamente mapeados a esquemas de tipado seguro con Zod.

💡
¿Por qué Stagehand? Permite programar la automatización mediante directivas en lenguaje natural (ej. "Haz clic en el enlace de la categoría Science"). La IA se encarga de analizar el DOM, calcular cuál es el elemento correcto y disparar el evento del navegador correspondiente.

📦 Requisitos e Instalación

Para ejecutar correctamente este proyecto en tu entorno local, sigue esta guía de instalación rápida paso a paso:

  • Verificar versión de Node.js (se requiere ^20.19.0 o >=22.12.0)
  • Instalar las dependencias base en el proyecto (npm install)
  • Obtener una Clave API de Gemini desde Google AI Studio
  • Crear archivo .env en la raíz del proyecto
  • Configurar la Clave API de Gemini en el archivo .env

Comandos de Instalación

Ejecuta este comando en la raíz del proyecto para instalar todas las dependencias necesarias:

Terminal (Bash / PowerShell) 
# Instalar todas las dependencias listadas en el package.json
npm install

# Instalación alternativa directa de paquetes mínimos:
npm install @browserbasehq/stagehand dotenv zod @ai-sdk/google

🔑 Configuración del Entorno

El proyecto hace uso de variables de entorno administradas a través del paquete dotenv. Es estrictamente necesario crear un archivo de configuración para almacenar las credenciales de los proveedores de LLM.

⚠️
Atención: Nunca subas el archivo .env a repositorios públicos como GitHub. El archivo de entorno contiene tu credencial de pago de API.

Estructura del archivo .env a crear en la raíz del directorio:

.env 
# Clave de API de Google Gemini (Obtenible en Google AI Studio)
GOOGLE_GENERATIVE_AI_API_KEY=TU_CLAVE_DE_API_DE_GEMINI_AQUI

🕹️ Explicación Detallada de Scripts

Cada script aborda un nivel diferente de complejidad y una funcionalidad clave de Stagehand. A continuación se presenta un desglose técnico de cada uno de ellos:

📖 1. Demostracion.mjs (Books to Scrape)

Este script contiene un flujo extendido y es ideal para entender los 4 pilares operativos que ofrece Stagehand:

  • Observe: Analiza visual y estructuralmente la página para buscar categorías de libros (sin selectores XPath/CSS manuales).
  • Act: Realiza una cadena de interacciones físicas consecutivas en el navegador:
    • Navegación: Hace clic en la categoría "Science".
    • Interacción: Hace clic en el título del tercer libro listado.
    • Carrito: Scrollea y hace clic en "Add to basket" para el segundo libro.
    • Navegación: Regresa a la página principal del catálogo.
  • Extract: Extrae información de la página mapeándola a un esquema tipado de Zod: 
    const esquemaInventario = z.object({
  productos: z.array(
    z.object({
      titulo: z.string(),
      precio: z.string(),
      enStock: z.boolean(),
    })
  )
});
    Los datos resultantes se vuelcan automáticamente en el archivo inventario_extraido.json.
  • Evals (QA Inteligente): Implementa lógica de aserciones dinámicas para validar que el texto del carrito contenga elementos agregados, determinando si el test de QA fue superado.
📷
El script genera evidencias automáticas de cada fase en formato .png dentro del directorio evidencias/ utilizando una marca de tiempo para evitar colisiones.
🤖 2. agente.mjs (Agente Autónomo)

Demuestra el poder del modo **Agente** en Stagehand. A diferencia del flujo guiado paso a paso, aquí delegamos la toma de decisiones al propio modelo: 

const agent = stagehand.agent();
await agent.execute("Navigate to the 'Poetry' category");

El agente inteligente observa la página inicial, calcula el camino para llegar a la categoría "Poetry", interactúa de forma autónoma con los elementos y finaliza cuando ha cumplido la directiva planteada.

🛒 3. amazon.mjs (Amazon Gaming Laptop Scraper)

Muestra la viabilidad de Stagehand en una plataforma e-commerce altamente compleja y protegida como **Amazon**.

  • Eliminación de Ruidos: Lanza una orden para cerrar diálogos intrusivos irrelevantes de ubicación o cookies.
  • Búsqueda: Escribe "laptop gaming" en el buscador y presiona la tecla Enter.
  • Extracción Semántica: Extrae el nombre, precio, rating de estrellas y especificaciones clave (como procesador, RAM) mapeando a un esquema Zod. Vuelca el resultado en laptop_amazon.json.
⚠️
Medidas Antibots de Amazon: Dado que corre localmente en tu red, Amazon puede presentar pantallas CAPTCHA o bloquear la navegación de Playwright. Se recomienda utilizar un modelo con mayor capacidad de razonamiento (google/gemini-2.5-pro) y configurar retardos amplios para imitar comportamientos humanos.

🛠️ Solución de Problemas Comunes (Troubleshooting)

💡
Error de Credenciales / API Key Inválida:

Asegúrate de que el archivo se llame exactamente .env y se encuentre en la raíz del proyecto. Si estás usando PowerShell, verifica la codificación del archivo (debe ser UTF-8 sin BOM).

💡
Error de Timeout (Tiempo de espera agotado):

Si tu conexión es lenta o las páginas demoran en cargar, edita Demostracion.mjs e incrementa el timeout del Stagehand (ej: timeout: 60000).

💡
Bloqueo Antibot en Amazon:

Amazon tiene protecciones robustas contra Playwright Headless. Prueba cambiando la propiedad del modelo a google/gemini-2.5-pro o bien utiliza un navegador persistente con datos de usuario reales, o corre la automatización usando los servicios de la nube de Browserbase.
